# Defined Workflow for Product Demo Video Production

## Purpose

This workflow standardises how product demo videos are created so the process is repeatable, faster to execute, and easier to refine over time.

It is designed to reduce variation, improve reuse, and separate:
- research
- scripting
- asset creation
- editing
- review
- learnings

At every stage, work must be reviewed by relevant colleagues before moving on.

---

## Core principle

Build a system, not one-off videos.

Each project should produce:
- a finished video
- reusable scripts
- reusable visual assets
- reusable learnings

---

## Standard workflow

### 1. Gather context
Collect the minimum information needed to understand the product, audience, and story.

Inputs may include:
- meetings with product or analyst team
- existing demos
- Miro boards
- rough notes
- technical walkthroughs
- previous scripts
- customer context

**Review checkpoint:** confirm understanding, unknowns, and scope with relevant colleague(s).

---

### 2. Define the story structure
Set the narrative structure before building visuals or editing.

Default structure:
1. Problem setup
2. Negative user story
3. Solution introduction
4. Demo segment
5. Under-the-hood segment
6. Positive user story
7. Outro / closing message

**Review checkpoint:** agree structure, section order, and ownership before scripting.

---

### 3. Write rough script
Write a rough script using the agreed structure.

At this stage:
- focus on meaning, not polish
- keep sections modular
- separate reusable core sections from audience-specific sections

Use AI to refine rough wording into the standard script style.

**Review checkpoint:** script must be reviewed for technical accuracy, clarity, and fit with the intended audience.

---

### 4. Create storyboard or slide MVP
Turn the script into a simple visual structure.

This can be:
- rough storyboard sketches
- simple slides
- placeholder frames

Purpose:
- test flow quickly
- align script and visuals
- expose weak transitions early

**Review checkpoint:** confirm the visual flow supports the script before asset creation.

---

### 5. Build visual assets
Create the minimum assets needed to prove the concept, while aiming for reuse.

Examples:
- backgrounds
- UI mockups
- intro/outro cards
- device illustrations
- diagram frames
- transparent PNG assets

**Review checkpoint:** confirm style, realism, and consistency before animation or editing.

---

### 6. Edit MVP
Combine script, visuals, voiceover, UI, and motion into a first working cut.

Focus on:
- pacing
- clarity
- alignment between narration and visuals
- whether the story lands

**Review checkpoint:** MVP must be reviewed internally before any external sharing.

---

### 7. Refine and finalise
Apply feedback, reduce unnecessary variation, and improve only what strengthens the standard.

At this stage:
- simplify
- standardise
- remove anything that does not improve clarity or reuse

**Review checkpoint:** final approval before client or stakeholder delivery.

---

## Rule for every stage

No stage is complete until it has been reviewed by the relevant colleague(s).

Review should check:
- technical accuracy
- storytelling clarity
- visual clarity
- strategic fit
- reuse potential
